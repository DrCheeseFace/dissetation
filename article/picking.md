15. Edge AI vision system

Short Description: Scene understanding and prompt reaction to an interesting event is a
critical feature for any time critical computer vision systems for safe and reliable interaction
with the environment. The deployment scenarios include a range of applications such as
mobile robotics, autonomous cars, mobile and wearable devices or public space
surveillance (airport / railway station). Modern vision systems which play a significant role in
such interaction process require higher level scene understanding with ultra-fast processing
capabilities operating at extremely low power. This is achievable through heterogeneous
vision system design with efficient implementation of computer vision algorithms.
In this project the students will develop computer vision algorithms on a heterogeneous
embedded camera system.
Requirements: Python, Image processing, UG: CSC3831, PG:CSC8628, CS8645.
Anticipated outcomes: Vision system demonstrator.

36. Autonomous Racing Project

In this project students will develop algorithms that will form an integral
part of Newcastle University’s Formula Student Autonomous Racing Team’s vehicle software
stack. The typical autonomous driving pipeline is split into 3 parts: Perception, Planning and
Control; The algorithms developed during the project may be a suitably advanced approach
addressing challenges in one of these areas, however we are also interested in End-To-End
approaches, where advanced AI models are trained to convert raw sensor input directly into
vehicle control commands. Working initially in simulation and with small scale F1tenth
autonomous racing cars, the goal is to contribute to a competition-ready software stack that
can be deployed in a Formula Student Autonomous Vehicle.
Requirements: Python, Deep Learning, Image processing, CSC3831/CSC8628, member of
Newcastle Racing AI team.
Anticipated outcomes: Algorithm deployable on autonomous racing car


30. Building Deep Learning NAS Benchmarks

One of the key concepts of Deep Learning is to understand which neural
network is best for a given dataset. To do this requires producing a benchmark where you
generate a large set (1000’s) of neural networks, train them and then provide a set of results
with the different network and results achieved. To do this is very labour intensive and can
require a lot of user interaction to build the networks and run all the training runs. This project
will develop a toolkit which will automate this process.
Requirements: Machine Learning, Deep Learning
Anticipated outcomes: A toolkit for producing NAS benchmarks.


53. Large Touch Screen Visualization for Exploration and

Communication of complex data to the general public and for decision
making is a major challenge in data science across a range of domains. Gaining of insights
and understanding of complex relationships in data, as well as learning and memorability,
may often increase through the ability to explore and interact directly with the data. Tools
such as large touch screens have potential to support communication and discussion of
findings in complex data, leading to further data driven insights and hypothesis generation.
This project will design and implement a visualization prototype with touch screen interaction
functionality using the large display in the USB Decision Theatre, addressing one of the
following challenges:
- Exploration of the Ocean Microbiome using the Tara Ocean’s data set
- Profiling of UK Crop data using a dataset from the UK Crop Microbiome Cryobank
- Exploration of missingness structures in incomplete data
Requirements: CSC3833 Data Visualization and Visual Analytics (for UG), or one of
CSC8626/CSC8636 (for PGT). Good programming skills. Python and/or JavaScript (D3),
understanding of basic computer graphics is beneficial.
Anticipated outcomes: Visual analytics prototype tools, novel implementation of
visualization methods. Improved programming, data analytics and visualization skills.

31. One network to rule them all (not)

Short Description: It is widely assumed that there is no single Deep Learning network which
is optimal for all datasets. This is often referred to as the ‘no free lunch’ case. However, no-
one has definitively shown this to be true. This project will work to demonstrate this by
identifying a network which is optimal for one dataset and show that it is not optimal on
another dataset.
Requirements: Python, Deep Learning
Anticipated outcomes: A set of examples of where the ‘no free lunch’ case is true



14. Modelling and optimising microgrid systems

Short Description: A microgrid is a local energy system that is made up of a hybrid of
distributed renewable resources, storage and cross-sectoral demands (electricity, heat and
mobility). While it provides access to affordable and clean energy, there is an industry need to
design a microgrid at least cost facing the demand of a local community. This project aims to
develop software functions that model a microgrid system of industry interests and optimise
the size of a relevant component (e.g. battery). Suitable energy modelling software plus
optimisation packages, will be investigated and exploited for the development.
Requirements: Knowledge of optimisation paradigms and energy systems, good
programming skills
Background reading: Rahmani, R., et al. Modelling and optimisation of microgrid
configuration for green data centres: A metaheuristic approach. Future Generation Computer
Systems, vol. 108, pp.742-750, 2020.
Anticipated outcomes: Software solutions for microgrid design.
